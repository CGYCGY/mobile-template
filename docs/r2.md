# Cloudflare R2 — File Uploads

R2 is the storage backend for user uploads (avatars, attachments, anything that doesn't belong in Convex). Convex actions mint presigned PUT URLs; the mobile client uploads directly to R2 over HTTPS; Convex records the resulting object key.

This split is deliberate: large blobs never traverse Convex, the client never sees R2 credentials, and a failed upload doesn't leave a half-written row.

## 1. Create the bucket

Cloudflare dashboard → R2 → **Create bucket**. Pick a name; you'll reference it as `R2_BUCKET`. Region is automatic.

If the bucket needs to serve files publicly (avatars on a CDN-cached URL), enable a public access domain — either `r2.dev` (rate-limited, fine for dev) or a custom domain. Note the base URL; you'll set it as `R2_PUBLIC_BASE_URL`.

## 2. Generate an R2 API token

Cloudflare dashboard → R2 → **Manage R2 API Tokens** → **Create API Token**.

- Permissions: **Object Read & Write**.
- TTL: pick something — these tokens are long-lived by default.
- Specify bucket: the one you just created. Don't grant account-wide access.

After creation, Cloudflare shows three values **exactly once** — copy them now:

- **Account ID** — `R2_ACCOUNT_ID`
- **Access Key ID** — `R2_ACCESS_KEY_ID`
- **Secret Access Key** — `R2_SECRET_ACCESS_KEY`

If you close the page without copying, you have to regenerate.

## 3. Set Convex env vars

R2 credentials live in Convex (never in the mobile bundle — these are server keys).

```bash
bunx convex env set R2_ACCOUNT_ID <your-account-id>
bunx convex env set R2_ACCESS_KEY_ID <key>
bunx convex env set R2_SECRET_ACCESS_KEY <secret>
bunx convex env set R2_BUCKET <bucket-name>
bunx convex env set R2_PUBLIC_BASE_URL <https://cdn.example.com>   # optional, only if public
```

You can also list them in `.env.local` under the "Convex-only env vars" section and run `just env-sync` — the recipe pushes any key prefixed `R2_` into Convex automatically.

## 4. Bucket CORS

Mobile clients have no browser origin, so CORS for the mobile-only flow is permissive. If the same bucket is also used by the web app, tighten `AllowedOrigins` to your domains.

```json
[
  {
    "AllowedOrigins": ["*"],
    "AllowedMethods": ["GET", "PUT", "HEAD"],
    "AllowedHeaders": ["*"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
```

Apply via the Cloudflare dashboard → bucket → **Settings → CORS Policy**, or with `wrangler r2 bucket cors put`.

## 5. Upload flow

```
mobile client                  Convex action            R2
─────────────                  ─────────────            ───
pick a file (camera/library)
hash + size + content-type
                ──── api.r2.requestUpload(args) ───→
                                  validate args
                                  presign PUT URL
                                  return { url, key, headers }
                ←── { url, key, headers } ─────────
PUT file body ──────────────────────────────────→
                                                  store
                ──── api.r2.confirmUpload(key) ───→
                                  persist key on user row / asset table
```

Key properties of this flow:

- The mobile client never sees R2 credentials. The presigned URL is single-use and short-lived (15 min in `convex/r2.ts`).
- The upload itself bypasses Convex — multi-MB files don't burn Convex action time or hit Convex size limits.
- `confirmUpload` is what makes the file "exist" from the app's perspective. If the client crashes between PUT and confirm, the R2 object is orphaned (clean these up with a lifecycle rule or a periodic job).

## 6. Code references

- `convex/r2.ts` — `requestUpload` and `confirmUpload` actions, plus the AWS SDK S3 client configured for R2.
- `lib/r2/upload.ts` — the mobile-side helper. Takes a `file://` URI from `expo-image-picker`, calls `requestUpload`, PUTs to the presigned URL with `expo-file-system`, then calls `confirmUpload`.

R2 speaks the S3 API. The Convex action uses `@aws-sdk/client-s3` + `@aws-sdk/s3-request-presigner` with the R2 endpoint (`https://<account>.r2.cloudflarestorage.com`). Anything you can do with S3, you can do with R2 — most existing S3 SDK code Just Works.
